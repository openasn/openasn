# enterprise_gateway.txt — ASNs of corporate/school secure web gateways (SWG).
#
# Effect: sets flags bit 11 (enterprise_gw). Classification:
# :enterprise_gateway (precedence rule 7 — ABOVE the hosting rules on
# purpose). Traffic from these ASNs is humans-at-work behind Zscaler-style
# egress, not anonymizer infrastructure. Blocking them blocks entire
# companies and school districts; the whole point of this class is to stop
# that false positive (the iboss case is documented by is-vpn's maintainer:
# https://github.com/josephrocca/is-vpn — school gateways misread as VPNs).
#
# Curation bar: ASN registered to a recognized SWG/SSE vendor whose product
# proxies customer web traffic through vendor-operated egress. Evidence:
# ipverse as-metadata org description
# (https://raw.githubusercontent.com/ipverse/as-metadata/master/as.json),
# checked 2026-07-04, plus the vendor's public product line.

# --- Considered and EXCLUDED (2026-07-08 audit; keep out unless the picture changes) ---
# - Cloudflare (AS13335 + siblings): primarily a CDN / content network (verdict :hosting,
#   D-ENRICH-1). Cloudflare Gateway/WARP egress is not separable from the CDN at ASN level
#   and the dominant use is content. NOT enterprise_gateway.
# - Palo Alto Networks own ASNs (AS2763 / 54538 / 64280 / 396421): the vendor's CORPORATE
#   networks, not customer egress — Prisma Access egresses from hyperscaler clouds, not
#   these ASNs. => :business (D-ENRICH-1), not enterprise_gateway.
