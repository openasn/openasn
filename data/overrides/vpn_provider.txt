# vpn_provider.txt — ASNs operated by / dedicated to consumer VPN & anonymizer services.
#
# Effect: sets flags bit 9 (vpn_provider). Classification: any IP announced by
# these ASNs gets verdict :vpn (precedence rule 6) unless a higher-precedence
# overlay (relay/tor) matched first. This is the ASN-level complement to the
# X4BNet range overlay: it covers newly-announced prefixes and ALL of the
# provider's IPv6 space, which the (IPv4-only) range overlay cannot.
#
# Curation bar: the ASN must be either (a) hand-curated as VPN infrastructure
# by X4BNet (first-party, MIT), or (b) registered to an organization whose
# primary business is a consumer VPN/anonymizer product, evidenced by the
# ipverse as-metadata description (CC0). Mixed-use hosting providers whose
# customers *include* VPNs (e.g. Datacamp/CDN77's parent AS60068) do NOT
# belong here — the range overlay handles those selectively.
#
# COUNTER-EXAMPLE (do not add): AS27683 "VPN de Mexico S.A. de C.V." — a
# corporate connectivity telco ("VPN" as in enterprise links), not an
# anonymizer. Keyword matches are candidates, never members.
#
# Line format: AS<number>  # <org> — <evidence>. src: <url> (<date checked>)
